
## 📘 大模型训练、微调、部署

### ✅ 总体说明

* **作业目标**：让学员在 3 天内完成一次 **从数据 → 微调 → 部署 → 评估** 的完整闭环
* **提交形式**：

  * Day 1 / Day 2：过程型作业（配置、截图、实验记录）
  * **Day 3：实战项目，提交最终成果**
* **分层要求**：

  * 🟢 无 Python 基础：以平台配置 + 理解流程为主
  * 🔵 有 Python 基础：以代码实现 + 可复现实验为主

---

### 🗓 Day 1 作业：大模型训练与微调基础（理解 + 上手）

#### 🎯 作业目标

* 理解大模型训练流程、迁移学习与 LoRA 微调原理
* 成功跑通一次 **“微调配置 → 启动训练 → 产出模型”** 的最小闭环



### 📝 作业内容

#### 【所有学员必做】

1. 选择一个开源模型（任选其一）：
   * LLaMA3-7B
   * Qwen3-4B/8B
2. 明确以下问题（文字说明即可）：

   * 原始模型 vs 微调模型的区别是什么？
   * LoRA 微调主要解决什么问题？
   * 本次微调使用的数据类型是什么（指令 / 对话 / 分类等）？



##### 🟢 无 Python 基础（低代码 / 可视化）

* 使用 **Hugging Face / ModelScope / 可视化训练平台**：

  * 选择模型
  * 上传或使用示例数据集
  * 配置 LoRA 参数（rank、learning rate 等）
  * 启动微调任务

📦 **提交物**

* 微调配置截图（模型、数据、LoRA 参数）
* 训练启动成功截图
* 300 字左右理解总结（“我理解的大模型微调流程”）



##### 🔵 有 Python 基础（代码方式）

* 使用 **LlamaFactory 或等价工具**

  * 编写或修改微调配置（YAML / Python）
  * 成功启动微调脚本
  * 能看到 loss 下降或训练日志

📦 **提交物**

* 微调配置文件
* 训练日志片段
* 简要说明：本次微调目标 + 数据说明

---

### 🗓 Day 2 作业：模型部署与性能评估（对比 + 量化）

#### 🎯 作业目标

* 学会将模型部署为可推理服务
* 能 **对比原始模型 vs 微调模型的效果与性能**


### 📝 作业内容


1. 完成一次模型推理测试：

   * 原始模型
   * 微调后模型
2. 设计 **至少 5 条测试样例**
3. 对比：

   * 输出质量（主观）
   * 响应速度 / 显存占用（客观）

##### 🔵 有 Python 基础

* 使用 **vLLM / Transformers**

  * 编写推理脚本
  * 使用 vLLM 加速推理
  * 监控 GPU 显存 / 利用率
* 编写一个简单 benchmark：

  * 准确率 / 命中率 / 人工打分均可

📦 **提交物**

* 推理脚本
* benchmark 设计说明
* 性能对比表（文本或表格）

---

### 🗓 Day 3 作业（实战）：全流程综合案例【最终交付】

> 🚀 **这是最终实战作业，需要提交完整成果**



#### 🎯 实战目标

独立完成一次 **“业务数据 → 模型微调 → API 部署 → 效果验证”** 的完整项目。



#### 🧩 实战任务说明（统一主题）

📌 **实战场景（任选其一）**

* 企业 FAQ / 客服问答
* 文档分类 / 意图识别
* 垂直领域问答（法律 / 医疗 / 金融 / 内部制度）


#### 🛠 实战任务拆解

##### ① 数据阶段

* 收集或构造业务数据
* 数据清洗
* 构建训练集（JSON / JSONL）

##### ② 训练阶段

* 使用 LoRA 微调开源模型
* 至少完成一次完整训练
* 评估模型效果

##### ③ 部署阶段

* 将模型部署为推理服务
* 提供 API 接口
* 测试多轮调用

##### ④ 验证阶段

* 对比原始模型与微调模型
* 给出结论与改进方向

#### 📦 最终提交成果（必须）

#### 🔵 有 Python 基础（完整项目）

* 项目代码仓库（或压缩包）：

  * `data/` 数据处理
  * `train/` 微调脚本
  * `deploy/` API 服务
* README：

  * 环境说明
  * 启动方式
  * 效果展示
* 实验结论总结
