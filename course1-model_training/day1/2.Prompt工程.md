## Prompt 工程


Prompt 工程（Prompt Engineering），通常翻译为提示工程，是一种设计、优化与迭代输入给大语言模型（如 ChatGPT、GPT-4/5 等）的提示语（prompt）的方法论和实践技巧，目的是让模型更准确、稳定、高质量地完成你想要的任务。

>一句话理解

- Prompt 工程 = 用“合适的提问方式”，把 AI 的能力最大化地“调教”出来。
- 同一个模型，不同 prompt，效果差异可能非常大。

### Prompt 是什么？

Prompt 指你给 AI 的全部输入内容，包括：**问题、指令、背景信息、示例、角色设定、输出格式要求**

例如：
```
“帮我写一份周报”
vs
“你是一名互联网产品经理，请用条列方式写一份 300 字以内的中文周报，重点突出数据与问题。”
```

第二个 prompt 的效果通常明显更好。

### Prompt 工程主要做什么？

Prompt 工程关注 4 件核心事情：

>1️⃣ 明确任务（Task）

- 你要 AI 做什么
- 写作 / 总结 / 翻译 / 编程 / 推理 / 角色扮演
    ```
    ❌ 模糊：“解释一下这个”
    ✅ 清晰：“用高中生能理解的方式解释什么是区块链，200 字以内”
    ```

>2️⃣ 提供上下文（Context）

- 给 AI 足够背景信息，减少猜测。
- 例如：目标读者是谁、使用场景、已知条件、限制条件

>3️⃣ 设定角色（Role）

- 让 AI “扮演”某种身份，能显著改变输出质量。
- 例如：“你是一名资深后端工程师”、“你是一位严格的论文审稿人”、“你是一位中文母语的法律顾问”

>4️⃣ 约束输出（Format & Style）

- 控制结果怎么呈现：字数、结构、语气、是否用表格 / 列表 / 代码块

### 常见 Prompt 工程技巧
>🔹 1. Few-shot（给示例）
- 通过示例教模型你想要的风格。

>🔹 2. Chain-of-Thought（引导思考）
- 让模型“分步骤思考”。
- 例如：“请一步一步分析，并给出最终结论。”

>🔹 3. 约束与检查

- “如果信息不确定，请明确说明不知道”
- “不要编造数据”
- “只基于我提供的内容回答”

>🔹 4. 迭代优化
- Prompt 工程不是一次写完，而是：写 → 测 → 改 → 再测

### 一个简单但高效的 Prompt 模板
```
你是谁（角色）：
你要做什么（任务）：
背景信息：
要求与限制：
输出格式：
```
